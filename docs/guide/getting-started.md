# 초기 연결 확인

> 이 섹션에서 무엇을 해결하나요?
>
> - 설치 직후 반드시 확인해야 할 연결 상태를 점검합니다.
> - 시리얼/MQTT/UI의 정상 동작 여부를 단계별로 검증합니다.
> - 문제 발생 시 어떤 로그를 먼저 봐야 하는지 정리합니다.

- 예상 소요 시간: **5분**
- 필수 준비물: 설치 완료된 Homenet2MQTT 인스턴스
- 완료 기준:
  - 시리얼 연결 정상
  - MQTT 연결 정상
  - UI 패킷 모니터 수신 확인

## 1) 시리얼 연결 확인

- `serial.path`가 실제 장치 경로(예: `/dev/ttyUSB0` 또는 `192.168.x.x:8899`)와 일치하는지 확인합니다.
- Add-on/Docker에서 장치 권한 또는 네트워크 접근이 허용되었는지 확인합니다.

## 2) MQTT 연결 확인

- `MQTT_URL`, 계정, 비밀번호를 재확인합니다.
- UI 상태 표시 또는 로그에서 브로커 연결 성공 메시지를 확인합니다.

## 3) UI 패킷 모니터 확인

- Web UI의 패킷 모니터 화면에서 RS485 수신 패킷이 들어오는지 확인합니다.
- 수신은 되지만 엔티티가 안 보이면 엔티티 타입 매핑을 재검토합니다.

## 4) 제조사 설정 점검

- 제조사 프로파일이 맞는지 [제조사 허브](../manufacturers/index.md)에서 다시 확인합니다.
- 제조사별 상세 예시는 각 제조사 문서를 참고하세요.
