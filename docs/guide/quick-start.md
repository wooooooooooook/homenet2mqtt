# 빠른 시작

> 이 섹션에서 무엇을 해결하나요?
>
> - 최소 단계로 Homenet Bridge (h2m) 를 실행하고 설정하는 방법을 안내합니다.
> - 사용자의 선호에 따라 Web UI를 사용하거나 YAML 파일을 직접 편집할 수 있습니다.

- 예상 소요 시간: **5~10분**
- 필수 준비물: RS485 장치, 실행 환경, 그리고 연동 방식에 따른 대상(MQTT 브로커 또는 Matter 허브)
- 완료 기준:
  - h2m 실행 환경 설치 완료
  - Web UI 또는 YAML 설정을 통한 기본 시리얼 통신 구성 완료
  - MQTT 연결 또는 Matter 페어링 완료 및 패킷 통신 확인

## 1) 설치 방법 선택

가장 먼저 h2m을 실행할 환경을 선택하고 설치합니다.

- Home Assistant 환경: [Add-on 설치 가이드](./install-addon.md)
- 일반 서버/NAS: [Docker 설치 가이드](./install-docker.md)

## 2) 연동 방식에 따른 연결 완료

설치를 마친 후, 선택한 연동 방식에 맞게 설정을 진행하고 정상 작동하는지 확인합니다.

### A. MQTT 연동 모드 (h2m-mqtt)
- Web UI의 상태 표시 섹션에서 **MQTT** 연결 상태가 정상인지 로그를 통해 확인합니다.
- Home Assistant에서 자동으로 기기들이 발견되는지 확인합니다.

### B. Matter 연동 모드 (h2m-matter)
- Web UI의 **Matter** 탭으로 이동합니다.
- [Matter 연결 가이드](./matter-connection.md)를 참고하여 스마트홈 플랫폼(애플 홈, 스마트싱스 등)에 기기를 추가합니다.

## 3) 공통 연결 확인 및 문제 해결

- 웹 UI 패킷 모니터에서 월패드 등 RS485 장치로부터 수신된 패킷이 정상적으로 표시되는지 확인합니다.
- 연결 실패 또는 작동하지 않는 경우 [트러블슈팅](./troubleshooting.md)으로 이동해 포트/권한 및 네트워크 설정을 점검하세요.
